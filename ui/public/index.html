How card data stays out of scope

1
Mint a Hosted Fields token

Server calls POST /api/hosted-fields/token with the merchant terminal. The browser never sees the API token.

2
Customer enters the card in Kadima's iframe

HostedFields.create() renders isolated fields. The PAN goes straight to Kadima — never to your backend.

3
Kadima processes & tokenizes

Auth or sale happens client-side; an externalId carries your order/session id.

4
You reconcile via webhook

transaction/create arrives signed; the backend matches it by externalId and records the payment.
